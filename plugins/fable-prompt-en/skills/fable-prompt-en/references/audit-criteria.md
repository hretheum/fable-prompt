# Self-audit criteria (Stage 11)

## Small-model tells — look for these in the finished prompt

- **Chunking instructions** — the work plan broken into click-by-click instructions instead of
  milestones.
- **Re-pasted-context rituals** — repeating/re-reminding context over and over in the prompt body.
- **Role incantations** — "You are an expert in X..." at the start. A Whole-Job Handoff starts with the
  OUTCOME, not a role.
- **Excessive constraint (over-constraint)** — rigid numeric values/decisions imposed without
  justification, where there should be an open decision with a citation requirement. Exception: values
  actually drawn from an authoritative source (e.g. a manufacturer's reference schematic) — that's not
  excessive constraint, that's a citation.

If you find any of the above — rewrite before saving the file, not after.

## 9-field coverage map

Before saving, check each of the 9 fields mentally (or in conversation, as a table): is it COMPLETE
(specific, verifiable), or just MENTIONED (generic, "something needs to go there")? A field that's only
generically mentioned = go back and press for more, before considering the stage closed.

## Honest assessment — is it worth spending Fable 5 resources

Return to the Stage 1 assessment. Ask yourself directly: after gathering all the details, does the task
still look like it requires days/weeks of human work and real judgment, or has it shrunk to something
an ordinary model would do in a few minutes? Say this to the user directly — arguments for and against,
not just a recommendation. This isn't a formality to click through, it's the last chance to avoid
wasting Fable 5's resources on something that doesn't need it.
